# Dead Code Audit Report (myPlanet Android)

Date: 2026-03-25  
Branch: `deadcode-audit-cleanup`

## Scope and method

This audit covered:
- Kotlin/Java source in `app/src/main/java`
- XML resources in `app/src/main/res`
- Gradle module/plugin/dependency declarations in `settings.gradle`, `build.gradle.kts`, and `app/build.gradle`

Primary analysis commands:
- `rg`-based symbol/reference scans for classes and resource IDs
- targeted regex scans for strings/colors/drawables/layout references
- manual review for dependency/plugin usage in Gradle scripts

> Note: `./gradlew app:lintDefaultDebug` could not run successfully in this environment because the `org.gradle.toolchains.foojay-resolver-convention:1.0.0` plugin could not be resolved from configured repositories. Findings below are based on static repository scans.

---

## 1) Kotlin/Java dead code findings

### Confirmed dead code (removed)
1. **`ExamSubmissionUtils` object** (`app/src/main/java/org/ole/planet/myplanet/utils/ExamSubmissionUtils.kt`)  
   - No references to `ExamSubmissionUtils` or its `saveAnswer` function in `app/src/main/java`, manifests, or resources.
   - Appears superseded by direct submission/answer handling elsewhere.

### Reflection / dynamic-loading risk flags
1. **`DispatcherModule` (`@Module`, `@InstallIn`)**  
   - Not referenced directly by source symbol search, but **used by Hilt code generation** (dynamic/annotation processing path).
   - **Not removed**.

### Potential false positives
- Inner data classes/view holders and callback classes may appear "unused" by simple symbol scans due to lexical scoping and generated bytecode usage.
- Android components declared in manifest or instantiated by framework are frequently missed by naive direct-reference scans.

## 3) Gradle modules/plugins/dependencies dead code findings

### Module-level finding
- `settings.gradle` includes only `:app`; there are no extra Gradle modules to remove.

### Confirmed dead build logic (removed)
1. **Unused plugin alias** in `app/build.gradle`:
   - `alias(libs.plugins.legacy.kapt)`
   - No `kapt(...)` dependencies or kapt processors were declared.
2. **Unused kapt block** in `app/build.gradle`:
   - `kapt { correctErrorTypes = true }`
   - No longer needed with current KSP-based processors.
3. **Redundant Kotlin stdlib dependency**:
   - `implementation(libs.kotlin.stdlib)` removed (stdlib is provided by Kotlin plugin/toolchain).

### Reflection / dynamic-loading risk flags
- None for the removed Gradle entries.

### Potential false positives
- If future annotation processors are reintroduced without KSP support, `kapt` plugin/block may be needed again.
